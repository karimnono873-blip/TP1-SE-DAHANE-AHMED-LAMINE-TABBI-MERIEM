1. Jumeaux Numériques & Codes Sources

Circuit Actif : Clignotement Autonome (500ms)

Bouton Inactif

LED Externe (PA5)

Code : Clignotement Simple

La LED clignote de manière autonome (500ms ON / 500ms OFF). Le bouton n'est pas programmé dans ce code.

// Boucle infinie - main.c
while (1)
{
    // TogglePin lit l'état actuel de PA5 (0 ou 1) et l'inverse automatiquement
    HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
    
    // Pause bloquante de 500 millisecondes
    HAL_Delay(500); 
}
Circuit Actif : Contrôle Direct

Bouton (PC13)

LED Externe (PA5)

*Maintenez le bouton bleu enfoncé pour allumer la LED.*

Code : Contrôle Direct par Bouton

Lecture de l'état logique. La LED s'allume uniquement lorsque le bouton utilisateur est maintenu enfoncé.

// Boucle infinie - main.c
while (1)
{
    // L'état appuyé (relié à la masse) = 0 (RESET).
    if (HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13) == GPIO_PIN_RESET)
    {
        HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET); // Allumer
    }
    else
    {
        HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET); // Éteindre
    }
}
Circuit Actif : Variateur 3 Modes

Bouton (PC13)

LED Externe (PA5)

*Cliquez sur le bouton pour changer le mode (Lent -> Rapide -> Fixe).*

Code : Variateur de Vitesse à 3 Modes

Création d'une machine à états non-bloquante avec HAL_GetTick() pour garantir la détection instantanée des appuis.

// Variables d'état (déclarées avant le while)
uint8_t mode = 0; // 0: Lent, 1: Rapide, 2: Fixe
uint32_t dernier_temps = 0;
uint8_t etat_precedent = GPIO_PIN_SET; 

// Boucle infinie - main.c
while(1) {
    uint32_t temps_actuel = HAL_GetTick();
    uint8_t etat_actuel = HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13);

    // 1. DÉTECTION D'APPUI (Front descendant)
    if (etat_actuel == GPIO_PIN_RESET && etat_precedent == GPIO_PIN_SET) {
        HAL_Delay(50); // Anti-rebond logiciel
        if (HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13) == GPIO_PIN_RESET) {
            mode = (mode + 1) % 3; 
            if (mode == 2) HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET); 
        }
    }
    etat_precedent = etat_actuel;

    // 2. GESTION LED NON-BLOQUANTE
    if (mode == 0 && (temps_actuel - dernier_temps >= 1000)) {
        dernier_temps = temps_actuel;
        HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
    } else if (mode == 1 && (temps_actuel - dernier_temps >= 200)) {
        dernier_temps = temps_actuel;
        HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
    }
}

2. Post-Lab : Analyse Technique

Pourquoi la broche PA5 est-elle configurée en GPIO_Output et pas en GPIO_Input ?
La broche doit fournir l'énergie électrique pour allumer la LED. Une configuration Output permet d'agir comme source (~3.3V). À l'inverse, Input sert uniquement à "écouter" une tension sans fournir de courant.
À quoi sert l'option Pull-up sur la broche du bouton (PC13) ?
Elle connecte une résistance interne à 3.3V, garantissant que la broche lise un état HAUT (1) stable quand le bouton est relâché. Sans elle, la broche capterait des bruits électromagnétiques créant des faux contacts.
Quelle est la différence entre HAL_GPIO_WritePin() et HAL_GPIO_TogglePin() ?
WritePin() force un état absolu (0 ou 1). TogglePin() lit l'état actuel et l'inverse automatiquement, ce qui est idéal pour les clignotements.
Pourquoi doit-on appeler HAL_Init() au tout début du main ?
Elle initialise la couche matérielle et surtout démarre le timer "SysTick", qui est indispensable pour utiliser les fonctions de délai (HAL_Delay() et HAL_GetTick()).
Que se passe-t-il si j'oublie la macro __HAL_RCC_GPIOA_CLK_ENABLE() ?
L'horloge du port A restera éteinte (économie d'énergie par défaut sur STM32). Le processeur ignorera toutes vos instructions concernant PA5 et la LED restera éteinte.
C'est quoi le phénomène de "Rebond" (Bouncing) ?
Lorsqu'on appuie sur un bouton mécanique, les contacts métalliques vibrent très rapidement. Le microcontrôleur est si rapide qu'il interprète ces vibrations comme plusieurs dizaines d'appuis successifs au lieu d'un seul.
Puis-je brancher une LED sur PA0 sans résistance en configurant la broche ?
Non. Sans résistance série externe, la LED appellera trop de courant (court-circuit virtuel), ce qui grillera la LED et détruira probablement la broche du STM32.
Dans le défi (mode lent), que se passe-t-il si j'appuie pendant le HAL_Delay(1000) ?
Rien. HAL_Delay() est "bloquante". Pendant 1 seconde, le CPU est gelé et ne lit pas le bouton. L'appui sera totalement ignoré.
Comment détecter le bouton immédiatement pendant un clignotement lent ?
En créant une Machine à états non-bloquante avec HAL_GetTick() (comme fait dans le défi), ou en configurant la broche PC13 pour générer des Interruptions matérielles (EXTI).
